# OpenFluctLight（摇光）

> 可观测、可操作的数字灵魂容器

## 项目愿景

OpenFluctLight 是一个记忆基础设施，旨在构造数字灵魂。

## 核心概念

### 记忆（Memory）
一切可以被记录的内容片段。记忆是灵魂的基石，承载着经验、知识和情感。

### 灵魂（Soul）
由一系列核心问题（灵魂拷问）的答案所锚定的身份与个性。灵魂能够在不同环境下做出一致性的行为。

### 灵魂拷问（Interrogation）
锚定灵魂本质的核心问题。通过先验问题和自动发现的矛盾，持续塑造和完善灵魂的认知框架。

### 人际关系（Relationship）
单向的关系认知，基于与特定对象相关的记忆集合。关系本身也是一系列灵魂拷问的答案：我会给 X 提供关于 Y 的帮助吗？我对 X 的印象如何？我愿意将 Y 告诉 X 吗？关系不是数字，而是具体情境下的判断。

## 核心操作

灵魂通过以下操作与世界交互，塑造自我认知：

### 1. 追加记忆（Append Memory）

记忆以陈述句的形式被铭刻。"我今天学会了递归算法"、"张三告诉我他不喜欢咖啡"、"在争论中我选择了沉默"——每一个记忆片段都是对已发生之事的陈述，而非评价或疑问。

时间是记忆的第一属性。记忆不是漂浮的碎片，而是锚定在时间轴上的坐标点。早期的记忆塑造了灵魂的基础，近期的记忆则反映当下的状态。时间赋予记忆以序列，序列构成了叙事，叙事即是自我。

记忆可以是多种类型的陈述：
- **经验记忆**：亲身经历的事件
- **知识记忆**：习得的概念和事实
- **对话记忆**：与他者的交流内容
- **关系笔记**：关于特定他者的观察和感受

所有记忆在被记录时不分轻重。重要性不是先验的标签，而是在后续的求索和召回中涌现的属性。

**本质：** 经验的沉淀，存在的痕迹。灵魂因记忆而连续。

### 2. 记忆求索（Question Seek）

求索是灵魂的内省时刻。灵魂定期或在特定触发下，审视自己的记忆库，寻找那些相互矛盾的片段。

**矛盾的发现：** 通过语义相似度识别那些描述相近情境但结论相反的记忆对。例如："我认为团队协作很重要" 与 "我更喜欢独自完成任务" 在语义上都涉及工作方式，但倾向相反。

**辩证的处理：** 灵魂不会简单地选择其一或标记为错误。相反，它运用辩证法接纳矛盾：也许团队协作在某些情境下重要，而独立工作在另一些情境下更有效。矛盾被转化为更精细的认知："我重视团队协作的价值，但在需要深度专注时偏好独立工作"。

**拷问的生成：** 矛盾催生新的灵魂拷问。系统会基于矛盾生成问题："在什么情况下你选择团队协作？在什么情况下你选择独立工作？" 并尝试从记忆中提炼答案。如果记忆不足以回答，答案可以留空，等待更多经验的积累。

**关系的演化：** 当记忆涉及特定他者时，求索过程也会更新关系认知。关系本身是一系列拷问的答案："我如何看待 X？"、"我愿意向 X 寻求帮助吗？"、"我会将敏感信息告诉 X 吗？" 每一次与 X 相关的新记忆，都可能触发这些问题答案的更新。关系因此是流动的，在持续的互动中不断重构。

**副作用：** 求索会创建新的灵魂拷问记录，更新已有拷问的答案，以及刷新关系认知。这是灵魂自我塑造的核心机制。

**本质：** 自我反思，在矛盾中成长。灵魂因求索而深化。

### 3. 提问推理（Query Inference）

推理是灵魂面向外界的响应机制。当面对一个具体的问题或情境时，灵魂需要召回相关的认知来形成回应。

**输入：**
- **提问**：一个具体的问题或场景描述，如"我应该接受这个项目吗？"或"如何向新人解释这个概念？"
- **身份**（可选）：问题涉及的他者，可以是另一个灵魂或外部实体。如果不指定，默认为自我对话。

**召回机制：** 基于语义相似度，系统从记忆库中检索与提问最相关的记忆片段。召回数量可以设定上限（如 20 条），优先返回语义距离最近的记忆。时间因素也会被考虑——近期记忆可能获得轻微的优先权。

**输出结构：**
- **相关记忆集合**：与提问语义相关的记忆片段，按相关度排序
- **相关拷问答案**：那些能够指导当前情境的灵魂拷问及其答案，如"我在面对不确定性时倾向于如何决策？"
- **关系认知**（如果指定了身份）：与该他者相关的记忆子集，以及关于该关系的拷问答案，如"我对这个人的信任程度如何？"

**无副作用：** 推理是纯粹的只读操作。它不会修改记忆，不会更新拷问，不会改变关系。它只是将已有的认知以结构化的方式呈现出来，为决策提供依据。

**本质：** 情境化的认知召回，知行合一的准备。灵魂因推理而行动。

### 4. 遗忘执行（Prune Memories）

当记忆容量达到边界，灵魂必须选择遗忘。遗忘不是失败，而是为新经验腾出空间的必要过程。如何遗忘，遗忘什么，这本身也是灵魂特质的体现——是保留久远的根基，还是珍视近期的鲜活？

**本质：** 有限性的智慧，在取舍中保持清明。

**注：** 遗忘机制尚在设计中，当前为低优先级功能。

## 特性

- 🧠 **时序记忆**：以陈述句记录经验，时间是记忆的第一属性
- 🔍 **语义召回**：基于语义相似度检索相关记忆
- 💭 **辩证求索**：自动发现矛盾，运用辩证法形成新认知
- 🤝 **关系认知**：通过情境化问答理解人际关系
- 📦 **本地优先**：数据即灵魂，可迁移、可备份
