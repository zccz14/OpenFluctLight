# OpenFluctLight（摇光）

> 可观测、可操作的数字灵魂容器

## 项目愿景

OpenFluctLight 是一个记忆基础设施，旨在构造数字灵魂。

## 核心概念

### 记忆（Memory）
一切可以被记录的内容片段。记忆是灵魂的基石，承载着经验、知识和情感。

### 灵魂（Soul）
由一系列核心问题（灵魂拷问）的答案所锚定的身份与个性。灵魂能够在不同环境下做出一致性的行为。

### 灵魂拷问（Interrogation）
锚定灵魂本质的核心问题。通过先验问题和自动发现的矛盾，持续塑造和完善灵魂的认知框架。

### 人际关系（Relationship）
单向的关系认知，基于与特定对象相关的记忆集合。关系本身也是一系列灵魂拷问的答案：我会给 X 提供关于 Y 的帮助吗？我对 X 的印象如何？我愿意将 Y 告诉 X 吗？关系不是数字，而是具体情境下的判断。

## 核心操作

灵魂通过以下操作与世界交互，塑造自我认知：

### 1. 追加记忆（Append Memory）
记忆以陈述句的形式被记录。每一个记忆片段都携带时间的印记，时间是记忆的本质属性。记忆不分轻重，它们共同构成了灵魂的经验基础。

**本质：** 经验的沉淀，存在的痕迹。

### 2. 记忆求索（Question Seek）
灵魂向内审视，在记忆的海洋中寻找矛盾。当发现两段记忆相互冲突时，灵魂不回避矛盾，而是运用辩证法接纳对立统一。矛盾本身成为新的认知，形成更深层的灵魂拷问与答案。

**本质：** 自我反思，在矛盾中成长。

### 3. 提问推理（Query Inference）
面对具体情境的提问，灵魂召回相关的记忆、已有的认知（灵魂拷问的答案），以及与特定他者的关系认知。这是灵魂对外界做出回应的基础。当提问涉及他者时，关系记忆被激活；当提问只关乎自我时，灵魂独自思考。

**本质：** 情境化的认知召回，知行合一的准备。

### 4. 关系演化（Relationship Evolution）
每一次与他者的互动都会在记忆中留下痕迹。这些痕迹不断积累，塑造着灵魂对他者的认知。关系不是静态的标签，而是持续演化的理解：我如何看待 X？我愿意与 X 分享什么？这些问题的答案随着新记忆的加入而更新。

**本质：** 关系是流动的认知，在互动中不断重构。

### 5. 遗忘执行（Prune Memories）
当记忆容量达到边界，灵魂必须选择遗忘。遗忘不是失败，而是为新经验腾出空间的必要过程。如何遗忘，遗忘什么，这本身也是灵魂特质的体现。

**本质：** 有限性的智慧，在取舍中保持清明。

**注：** 遗忘机制尚在设计中，当前为低优先级功能。

## 特性

- 🧠 **时序记忆**：以陈述句记录经验，时间是记忆的第一属性
- 🔍 **语义召回**：基于语义相似度检索相关记忆
- 💭 **辩证求索**：自动发现矛盾，运用辩证法形成新认知
- 🤝 **关系认知**：通过情境化问答理解人际关系
- 📦 **本地优先**：数据即灵魂，可迁移、可备份
