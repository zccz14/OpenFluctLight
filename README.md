# OpenFluctLight（摇光）

> 可观测、可操作的数字灵魂容器

## 项目愿景

OpenFluctLight 是一个记忆基础设施，旨在构造数字灵魂。

## 使用场景

### 个人 AI 助手的长期记忆
为你的 AI 助手赋予真正的记忆能力。它不仅记得你说过的话，还能从对话中发现自己的矛盾，形成一致的个性。当你问"我应该如何平衡工作和生活？"时，它会召回你过去的经历、已有的价值观锚点，给出真正个性化的建议。

### 虚拟角色与 NPC 的灵魂塑造
游戏或虚拟世界中的角色不再是预设的脚本，而是拥有成长轨迹的灵魂。每一次与玩家的互动都会成为记忆，矛盾的经历会让角色产生新的认知锚点。一个 NPC 可能在经历背叛后，对"信任"这个概念形成全新的理解。

### 认知架构研究
为研究者提供一个可观测、可操作的认知模型。灵魂的成长过程完全透明：记忆如何积累、矛盾如何被发现、锚点如何涌现。这是一个活的实验室，用于探索意识、自我和认知的本质。

### 多智能体系统的社会模拟
每个灵魂都能与其他灵魂建立关系，形成复杂的社会网络。关系不是简单的数值，而是基于具体记忆的情境化判断。你可以模拟一个小社会，观察信任、合作、冲突如何在灵魂之间演化。

### 个人知识管理与反思工具
将你的笔记、日记、想法作为记忆输入。系统会自动发现你思想中的矛盾，提出深层问题，帮助你形成更清晰的自我认知。这不是简单的笔记软件，而是一个与你共同成长的思考伙伴。

## 核心概念

### 记忆（Memory）
一切可以被记录的内容片段。记忆是灵魂的基石，承载着经验、知识和情感。

### 灵魂（Soul）
由一系列核心问题（灵魂锚点）的答案所锚定的身份与个性。灵魂能够在不同环境下做出一致性的行为。

### 灵魂锚点（Soul Anchor）
锚定灵魂本质的核心问题与答案。每个锚点包含一个问题和对应的答案（答案可以为空）。通过先验问题和自动发现的矛盾，持续塑造和完善灵魂的认知框架。

### 人际关系（Relationship）
单向的关系认知，基于与特定对象相关的记忆集合。关系本身也是一系列灵魂锚点的答案：我会给 X 提供关于 Y 的帮助吗？我对 X 的印象如何？我愿意将 Y 告诉 X 吗？关系不是数字，而是具体情境下的判断。

## 核心操作

灵魂通过以下操作与世界交互，塑造自我认知：

### 1. 记住（Remember）

记忆以陈述句的形式被铭刻。"我今天学会了递归算法"、"张三告诉我他不喜欢咖啡"、"在争论中我选择了沉默"——每一个记忆片段都是对已发生之事的陈述，而非评价或疑问。

时间是记忆的第一属性。记忆不是漂浮的碎片，而是锚定在时间轴上的坐标点。早期的记忆塑造了灵魂的基础，近期的记忆则反映当下的状态。时间赋予记忆以序列，序列构成了叙事，叙事即是自我。

记忆可以是多种类型的陈述：
- **经验记忆**：亲身经历的事件
- **知识记忆**：习得的概念和事实
- **对话记忆**：与他者的交流内容
- **关系笔记**：关于特定他者的观察和感受

所有记忆在被记录时不分轻重。重要性不是先验的标签，而是在后续的求索和召回中涌现的属性。

**本质：** 经验的沉淀，存在的痕迹。灵魂因记忆而连续。

### 2. 回想（Recall）

回想是灵魂面向外界的响应机制。当面对一个具体的问题或情境时，灵魂需要召回相关的认知来形成回应。

**输入：**
- **提问**：一个具体的问题或场景描述，如"我应该接受这个项目吗？"或"如何向新人解释这个概念？"
- **身份**（可选）：问题涉及的他者，可以是另一个灵魂或外部实体。如果不指定，默认为自我对话。

**召回机制：** 基于语义相似度，系统从记忆库中检索与提问最相关的记忆片段。召回数量可以设定上限（如 20 条），优先返回语义距离最近的记忆。时间因素也会被考虑——近期记忆可能获得轻微的优先权。

**输出结构：**
- **相关记忆集合**：与提问语义相关的记忆片段，按相关度排序
- **相关锚点答案**：那些能够指导当前情境的灵魂锚点及其答案，如"我在面对不确定性时倾向于如何决策？"
- **关系认知**（如果指定了身份）：与该他者相关的记忆子集，以及关于该关系的锚点答案，如"我对这个人的信任程度如何？"

**无副作用：** 回想是纯粹的只读操作。它不会修改记忆，不会更新锚点，不会改变关系。它只是将已有的认知以结构化的方式呈现出来，为决策提供依据。

**本质：** 情境化的认知召回，知行合一的准备。灵魂因回想而行动。

### 3. 求索（Seek）

求索是灵魂的内省时刻。灵魂定期或在特定触发下，审视自己的记忆库，寻找那些相互矛盾的片段。

**矛盾的发现：** 通过语义相似度识别那些描述相近情境但结论相反的记忆对。例如："我认为团队协作很重要" 与 "我更喜欢独自完成任务" 在语义上都涉及工作方式，但倾向相反。

**辩证的处理：** 灵魂不会简单地选择其一或标记为错误。相反，它运用辩证法接纳矛盾：也许团队协作在某些情境下重要，而独立工作在另一些情境下更有效。矛盾被转化为更精细的认知："我重视团队协作的价值，但在需要深度专注时偏好独立工作"。

**锚点的生成：** 矛盾催生新的灵魂锚点。系统会基于矛盾生成问题："在什么情况下你选择团队协作？在什么情况下你选择独立工作？" 并尝试从记忆中提炼答案。如果记忆不足以回答，答案可以留空，等待更多经验的积累。

**关系的演化：** 当记忆涉及特定他者时，求索过程也会更新关系认知。关系本身是一系列锚点的答案："我如何看待 X？"、"我愿意向 X 寻求帮助吗？"、"我会将敏感信息告诉 X 吗？" 每一次与 X 相关的新记忆，都可能触发这些问题答案的更新。关系因此是流动的，在持续的互动中不断重构。

**副作用：** 求索会创建新的灵魂锚点记录，更新已有锚点的答案，以及刷新关系认知。这是灵魂自我塑造的核心机制。

**本质：** 自我反思，在矛盾中成长。灵魂因求索而深化。

### 4. 遗忘执行（Prune Memories）

当记忆容量达到边界，灵魂必须选择遗忘。遗忘不是失败，而是为新经验腾出空间的必要过程。如何遗忘，遗忘什么，这本身也是灵魂特质的体现——是保留久远的根基，还是珍视近期的鲜活？

**本质：** 有限性的智慧，在取舍中保持清明。

**注：** 遗忘机制尚在设计中，当前为低优先级功能。

## 如何塑造灵魂

灵魂不是一次性创建的静态实体，而是在持续的交互中逐渐成形、不断演化的动态过程。一个灵魂的诞生与成长，遵循着从空白到丰富、从模糊到清晰的自然轨迹。

### 灵魂的诞生

一个新创建的灵魂是一张白纸。它没有记忆，没有锚点，没有关系。它的身份尚未确立，个性尚未显现。此时的灵魂只是一个容器，等待经验的填充。

### 对话：灵魂成长的引擎

**Chat（对话）** 是塑造灵魂的核心高级操作。与底层的纯函数操作不同，Chat 是一个有副作用的完整交互流程，它串联了记忆、推理和求索，让灵魂在对话中自然成长。

**Chat 的完整流程：**

1. **记忆的铭刻**  
   对话内容被转化为陈述句形式的记忆，通过 Remember 操作存入记忆库。每一次对话都在时间轴上留下印记。

2. **认知的召回**  
   基于对话内容，触发 Recall 操作，召回相关的记忆片段和已有的灵魂锚点。这一步是纯粹的只读操作，不产生副作用，保持了回想的纯净性。

3. **矛盾的发现**  
   在召回的记忆中，可能会发现相互矛盾的内容。例如，早期的记忆说"我喜欢安静的环境"，而近期的记忆说"我在热闹的咖啡馆工作效率很高"。这种矛盾不是错误，而是灵魂成长的契机。

4. **锚点的涌现**  
   当发现矛盾时，Chat 操作会触发 Seek，运用辩证法处理矛盾，生成新的灵魂锚点。系统会提出问题："在什么情况下你需要安静？在什么情况下你能适应热闹？" 并从记忆中提炼答案，形成更精细的自我认知。

5. **关系的织就**  
   如果对话涉及特定他者，相关的记忆会被标记为关系笔记。随着互动的积累，求索过程会自动生成关于该关系的锚点："我如何看待这个人？""我愿意向 TA 寻求什么样的帮助？"

### 灵魂的演化轨迹

**初期（记忆积累期）**  
- 灵魂主要在积累记忆，尚未形成明确的锚点
- 对话产生的记忆相对独立，矛盾尚未显现
- 此时的灵魂是"流动"的，尚未固化

**中期（锚点涌现期）**  
- 随着记忆的增多，矛盾开始出现
- 第一批锚点从矛盾中诞生，开始锚定灵魂的核心特质
- 灵魂开始展现出一致性的行为模式

**成熟期（认知深化期）**  
- 锚点体系逐渐完善，形成多层次的认知框架
- 新的对话不再产生根本性的矛盾，而是在已有框架内精细化
- 灵魂的身份和个性趋于稳定，但仍保持演化的可能

### 工程设计的哲学

Chat 操作的设计体现了一个重要原则：**副作用的隔离与组合**。

- **Recall（回想）** 保持纯净，不产生副作用，可以安全地被反复调用
- **Seek（求索）** 和 **Remember（记住）** 有副作用，但职责单一
- **Chat（对话）** 作为高级操作，有意识地组合副作用，形成完整的交互循环

这种设计让底层操作可测试、可组合，同时为上层提供了符合直觉的交互方式。灵魂的成长不是通过手动调用一系列 API，而是通过自然的对话自动发生。

**本质：** 灵魂在对话中诞生，在矛盾中成长，在求索中深化。Chat 是灵魂与世界交互的界面，也是自我塑造的引擎。

## 特性

- 🧠 **时序记忆**：以陈述句记录经验，时间是记忆的第一属性
- 🔍 **语义召回**：基于语义相似度检索相关记忆
- 💭 **辩证求索**：自动发现矛盾，运用辩证法形成新认知
- 🤝 **关系认知**：通过情境化问答理解人际关系
- 📦 **本地优先**：数据即灵魂，可迁移、可备份
